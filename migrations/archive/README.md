# Archived migrations

These migrations are kept for historical reference only. They are NOT in
`migrations/meta/_journal.json` and will NOT be applied by `drizzle-kit migrate`.

The current source of truth for fresh-DB recovery is
`migrations/0002_consolidated_snapshot.sql`, which captures the cumulative
effect of everything in this folder.

## How we got here

The drizzle journal originally tracked only the first two files (`0000_*` and
`0001_*`, generated by `drizzle-kit generate` in March). Subsequent schema
changes (`003_*` through `011_*`) were hand-written and applied directly via
the Supabase SQL editor — they never entered the journal. Slot `002` was
historically unused (file naming jumped from drizzle's 4-digit `0001` to the
hand-written 3-digit `003`), so PR-E (audit-2026-05-15) reclaims slot `0002`
for the consolidating snapshot.

## If you need to recover a fresh DB

Run `drizzle-kit migrate` against an empty Postgres. It will apply
`0002_consolidated_snapshot.sql`, which creates every table, index, and RLS
posture in one shot. Do NOT also apply files from this archive — they would
be no-ops thanks to `IF NOT EXISTS` everywhere, but they're not needed.
